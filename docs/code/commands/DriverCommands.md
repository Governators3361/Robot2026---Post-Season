# Driver Commands

<div class="field-section">
  <h3>CommandXStop</h3>
  <div class="field-col">
    <div class="field">
      <label>This command runs on a whileTrue button scheduler, and follows according coding principles. This command is quite simple, it puts the swerve wheels in an X formation, where every while is aligned, pointing outwards towards their respective corners. This allows the robot to brake, and be resistant to bumps. This is an essential feature for drivers, and must be on all future revisions of the robot.</label>
    </div>
  </div>
  <h3>CommandToggleTurbo</h3>
  <div class="field-col">
    <div class="field">
      <label>This command simply increases the rate limits for maximum translational and rotational speeds. This was used so much during the season that it became the norm, and instead rate limits were implemented (depending on when you are reading this) to prevent from brownouts. This really was a quick fix, don't feel like you are missing out on "speed" in the future, this shouldn't be necessary.</label>
    </div>
  </div>
</div>
